Σύντομη περιγραφή:
---------------
Server:
    - Ο server ορίζει τις global μεταβλητές που βρίσκονται στο globals.h. Αυτές είναι απαραίτητο να είναι global στην συγκεκριμένη υλοποίηση γιατί χρησιμοποιύνται από την συνάρτηση HandleCtrlC η οποία δεν μπορεί να πάρει ορίσματα.
    Οι υπόλοιπες μεταβλητές που χρειάζεται να μοιράζονται μεταξύ τους τα νήματα αλλά δεν είναι απαραίτητο να είναι global τοποθετούνται στην δομή Shared.
    - Η δομή shared ορίζεται και τα μέλη της αρχικοποιύνται από την main συνάρτηση, και όχι το master thread για να μπορούν να ελευθερωθούν κατά την λήξη του προγράμματος (θα επεξηγηθεί παρακάτω).
   -- Master thread:
        - Το master thread αρχικοποιεί το socket και περιμένει με την accept() συνδέσεις για να τις τοποθετήσει στον buffer. Ο μεγάλος αριθμός του πεδίου backlog της listen() (100) οφείλεται στο γεγονός ότι κατά τις δοκιμές το πρόγραμμα σε μεγάλα Input έπεφτε σε
        deadlock, άσχετο με το μέγεθος του buffer, καθώς ο server περίμενε στην accept() να έρθουν αιτήσεις οι οποιές είχαν σταλθεί από τον πελάτη άλλα είχαν διαγραφεί καθώς ξεπέρασαν τον αριθμό που ορίζει το πεδίο backlog.
        - Αφού δεκτεί κάποια αίτηση γράφει στον buffer της σύνδεση αν δεν είναι γεμάτος, αλλίως μπλοκάρεται.
        - Αν τυχόν πατηθεί ctrl+c αφού ενώ περιμένει να αδειάσει ο buffer, θα ακυρώσει την σύνδεση και θα κάνει κανονικά exit. Αν πατηθεί αφού περάσει το σημείο αυτό, θα γράψει την σύνδεση στον buffer και θα κάνει exit.
         Στην (πιο πιθανή) περίπτωση που πατηθεί ενώ περιμένει νέα σύνδεση στην accept() (οι πελάτες έχουν τερματίσει) το νήμα πρέπει να "σκοτωθεί" με τη βία καθώς δεν υπάρχει τρόπος να βγεί από την accept. Για αυτόν τον λόγο αναφέρθηκε παραπάνω
         ότι το πρόγραμμα είναι δομημένο έτσι ώστε τα resources που χρησιμοποιεί το master threaad να μην χρειάζεται να γίνουν free από το ίδιο.
   -- Worker threads:
        - Ξεκινούν κλειδώνοντας το mutex και ελέγχνοτας να υπάρχει σύνδεση τον buffer. Αν υπάρχει την αφαιρούν και μετακινούν τις εγγραφές μια θέση κάτω.
        - Αφού ελευθερώσουν το mutex και ειδοποιήσουν το master thread ότι δημιουργήθηκε χώρος στον buffer, επικοινωνούν με τον πελάτη με την σειρά μηνυμάτων που ορίζει η εκφώνηση. Αποθηκεύοντας το όνομα που λαμβάνουν.
        - Τα όνοματα που λαμβάνονται τοποθετούνται σε ένα hash table ώστε να μπορεί να γίνεται γρήγορα ο έλεγχος αν κάποιο όνομα έχει ήδη ψηφίσει.
        - Στην περίπτωση που έχει τερματίζεται η σύνδεση με το κατάλληλο μύνημα στον πελάτη, αλλιώς το όνομα τοποθετήται στο hash table και συνεχίζεται η επικοινωνία.
        - Αφού κατασκευάσει το string : [name] [last_name] [vote] κλειδώνει το file mutex για να γράψει στο log και τοποθετεί εκεί την εγγραφή του.
        - Τα mutexes συνοδεύονται από conditional variables για να αποφευχθεί το busy waiting.
        - Σε περίπτωση ctrl+c δίνεται σήμα μέσω του buffernotEmpty ώστε να προχωρήσει το επόμενο thread που περιμένει σύνδεση, να διαβάσει το flag, να κάνει signal στο επόμενο και να κάνει κανονικά exit.
        - Αν το thread έχει ήδη λάβει σύνδεση αφού ληφθεί το σήμα θα τελειώσει την επικοινωνία και εγγραφή και θα κάνει exit στο τέλος του loop.

    -- Stats.c:
        - Τα στατιστικά φτιάχνονται αφού ληφθεί σήμα εξόδου. Δημιουργεί έναν πίνακα από κόμματα, με το κάθε κόμμα να αναπαρίσταται από το όνομά του και τον αριθμό ψήφων που έχει κατα την διάρκεια της καταμέτρησης που ακολουθεί.
        - Αφου ανοιχτούν τα δύο αρχεία (log και stats) ξεκινά η γραμμή-γραμμή ανάγνωση του log. Κάθε γραμμή γίνεται tokenise για να διαβαστεί η τρίτη λέξη (όνομα κόμματος).
        - Αν το όνομα του κόμματος δεν υπάρχει στον πίνακα, τοποθετήται.
        - Στο τέλος της μέτρησης αναγράφεται στο αρχείο κάθε κόμμα με τις ψήφους που έχει καθώς και οι συνολικές ψήφοι.

Client:
        - Ο client ξεκινά αντιγράφοντας κάθε γραμμή του Input αρχείου ώστε να μπορεί να περαστεί στο αρμόδιο νήμα σαν όρισμα.
        - Οι γενικές πληροφορίες που χρειάζονται τα νήματα τοποθετούνται σε δομή shared αντίστοιχη με αυτή του server.
        - Το κάθε νήμα κάνει tokenise την γραμμή που του δώθηκε και επικοινωνεί με το αντίστοιχο νήμα του server όπως αναφέρεται στην εκφώνηση.



BASH SCRIPTS: cd bash_scripts
        Σημαντικό:
            * Δεν χρειάζεται να συμπεριληφθεί το path στο όνομα των αρχείων εισόδου-εξόδου, τα αρχεία τοποθετούνται αυτόματα στον κατάλληλο κατάλογο.
            * Στα μηχανήματα linux του τμήματος χρειάζεται το processLog.sh να εκτελεστεί ως " bash ./processLog.sh poll_log ". Σε τοπικό υπολογιστή με wsl εκτελείται χωρίς πρόβλημα με την εντολή που αναγράφεται παρακάτω.

create_input.sh:  ./create_input.sh parties.txt numlines
            Δημιουργεί τυχαίες ψήφους. Το αρχείο που δημιουργεί τοποθετήται στον κύριο κατάλογο και ονομάζεται πάντα inputFile.txt. Αν το αρχεία υπάρχει κάνει overwrite.

tallyVotes.sh:     ./tallyVotes.sh tallyResultsFile

            Χρησιμοποιεί associative arrays για να αποθηκεύσει αρχικά τις ψήφους (μια θέση για κάθε συνδυασμό ονόματος-επιθέτου) ελέγχνοτας για duplicates, και τις ψήφους ανά κόμμα.
            Αν το αρχείο tallyResultsFile υπάρχει ήδη γίνεται overwritten.

processLog.sh:      ./processLog.sh poll_log

            Ο κώδικας είναι παρόμοιος με την tallyVotes με την διαφορά ότι τώρα πρέπει να μπεί μπροστά από το όνομα του αρχείου που δίνεται το path για τον κύριο κατάλογο.
            Επίσης δεν χρειάζεται να γίνεται έλεγχος για διπλότυπα καθώς φροντίζει ο server να μην υπάρχουν και το αρχείο log προέρχεται από εκεί.

            Στο τέλος των tallyVotes.sh και processLog.sh γίνεται sort ώστε να μπορούν να συγκριθούν.
